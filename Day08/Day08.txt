1. redux-saga
    - redux-thunk 다음으로 가장 많이 사용되는 비동기 작업 처리 미들웨어
    - 다음 상황에서 redux-saga가 많이 사용됨
      기존 요청을 취소할 때
      특정 액션이 발생했을 때 다른 액션을 발생 시키거나, 
      API 요청 등 리덕스와 관계없는 코드를 실행시킬 때
      웹 소켓을 사용할 때
      API 요청 실패 시 재요청할 때
    - redux-saga는 재네레이터 함수를 사용한다.
      function* generatorFnc() {
        yield 1;
        yield 2;
        yield 3;
        return 4;
      }
    - redux-saga는 우리가 디스패치하는 액션을 모니터링해서 그에 따라
      필요한 작업을 따로 수행하도록 한다.
